# Переопределение методов класса Object

## GenericsAndEqualsByInterfaceExample

GenericsAndEqualsByInterfaceExample — точка входа для демонстраци примера использования обобщений из рабочей 
практики, когда есть несколько разных классов, репрезентующих одну сущность, но с небольшими отличиями.

Классы объединены общим интерфейсом, благодаря чему можно использовать один обобщеный метод 
для одинаковой обработки объектов любого из этих классов.

Кроме того, для сравнения объектов этих классов в них были переопределены методы equals и hashCode
с применением общего интерфейса, т.к. сравнение по придуманной бизнес-логике должно идти по общим
для этих классов полям.

Бизнес-идея:
- классы ServiceAvailable и ServiceConnected отвечают за представление доступной и подключенной услуги;
- есть несколько списков: список доступных услуг, список подключенных услуг, список интересующих бизнес кодов услуг;
- необходимо определить, какие услуги из списка кодов уже не востребованы, какие пользуются спросом, а какие уже в архиве, но еще есть последние потребители.



## ShallowAndDeepCopyExample
ShallowAndDeepCopyExample в пакете clone_demonstration — точка входа для демонстрации примеров поверхностного (shallow copy) 
и глубокого (deep copy) клонирования объектов. Глубокое клонирование — на основе переопределения метода clone, это классический подход.

Пример клонирование с использованием подхода конструктора копирования можно посмотреть в UserProcess.



## UserProcess

UserProcess — точка входа для домашней работы по теме методов класса Object.

**Задача 1**

Создать класс для описания пользователя системы. Переопределить в классе методы toString,hashcode и equals. 
Создать несколько экземпляров класса с одним и тем же значением полей и сравнить с помощью метода equals.

**Задача '*'**

Создать программу для реализации поверхностного и глубокого клонирования объекта класса User. 
Пусть на вход программе будет передаваться тип операции клонирования (поверхностное клонирование или глубокое), 
а также id юзера для клонирования.

Глубокое клонирование выполнено через конструктор копирования.
